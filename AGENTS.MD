# Agent Rules

本檔案定義所有後續 agent 在此 repository 工作時必須遵守的流程。

## 遠端版本檢查

在新對話開始的時候，必須先確認目前本地分支與其 upstream 遠端分支是否一致。

建議流程：

1. 執行 `git fetch origin` 更新遠端追蹤資訊。
2. 執行 `git status --short --branch` 檢查目前分支、upstream 與工作區狀態。
3. 執行 `git rev-list --left-right --count HEAD...@{u}` 檢查本地與遠端差異。

判讀方式：

- 輸出為 `0 0`：本地與遠端版本一致，可以繼續工作。
- 第一個數字大於 `0`：本地有尚未推送的 commit。
- 第二個數字大於 `0`：遠端有本地尚未取得的 commit。
- 任一數字不為 `0` 時，必須先詢問使用者要如何處理，不可自行 merge、rebase、pull、push 或覆蓋。

如果目前分支沒有 upstream，必須先詢問使用者要追蹤哪個遠端分支。

## 修改紀錄

專用修改紀錄檔案為 `MODIFICATION_LOG.md`。

每次準備將修改後的版本推到遠端前，必須先在 `MODIFICATION_LOG.md` 新增一筆紀錄，內容至少包含：

- 日期
- 分支
- 變更摘要
- 影響的檔案或範圍


禁止在未更新 `MODIFICATION_LOG.md` 的情況下推送修改到遠端。

更新時越上面修改紀錄是越新的，舊的會被往下推

## 保護既有變更

工作區若存在非本次任務造成的修改、刪除或未追蹤檔案，必須保留並避開；除非使用者明確要求，禁止復原、覆蓋或刪除這些變更。
